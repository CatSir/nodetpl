nodetpl.config(options)

Configuration nodetpl basic parameters

config is not necessary, generally you do not need to do any configuration

The following are examples of parameters that can be configured:

nodetpl.config({
  base: 'http://www.nodetpl.com/static/tpl/',   // Forced specify a template root directory
  openTag: '<?',
  closeTag: '?>',
  strict: true,
  map: function(str) {
    return str;
  }
});
  • base: root path where the template can be omitted. If you configure the parameters, use the .get() method to fill in the path will be relative to the path to find
  • openTag: start tag of template delimiters, by default: <?
  • closeTag: end tag of template delimiters, by default: ?>
  • strict: wither use strict mode, by default: true
  • map: pass an analytic function, and to parse other grammar, such as PHP, JSP, etc

If you do not enable strict mode, then in the template "@" can be omitted, eg:

<h2><?=@title?></h2>

If not under the "strict mode" can be written as:

<h2><?=title?></h2>

But, with the development of the ES6, we do not recommend this.

Here most of the default values can be used, so that this method can be omitted. Only in a handful of scenes, you can make adjustments according to your own needs.

Custom syntax: powerful map() method

nodetpl allows you to pass any grammar you like, you can according to need, pass the map() method, tell nodetpl how to resolve these unconventional statements, the following is a simple example:

If you have the following template:

<div>
  <?foreach ($array as $key => $value):?>
    hello, <?=$key?> <?=$value?>
  <?endforeach?>
  <?for($i=0; $i<=10; $i++):?>
    hello, <?=$i?>
  <?endfor?>
  <?if($age < 20):?>
    hello, young man.
  <?endfor?>
</div>

Obviously, nodetpl don't know how to deal with this kind of grammar, but we can through the map() tell nodetpl, meet with these statements, how to deal with:

nodetpl.config({
  strict: false,
  map: function(str) {
    // foreach => arr.forEach
    str = str
      .replace(/foreach\s*\(\s*(\$array)\s+as\s+(\$key)\s*=>\s*(\$value)\s*\):/g, function(all, arr, key, value) {
        return arr.replace(/^\$/g, '') + '.forEach(function(' + value.replace(/^\$/g, '') + ', ' + key.replace(/^\$/g, '') + ') {';
      })
      .replace(/endforeach/g, '});');
    // for
    str = str
      .replace(/for\s*\(\s*([^;]+)(.*)\):/g, function(all, vars, conditions) {
        return 'for(var ' + vars.replace(/\$/g, '') + conditions.replace(/\$/g, '') + ') {';
      })
      .replace(/endfor/g, '}');
    // if
    str = str
      .replace(/((?:else\s+)?if)\s*\((.*)\):/g, function(all, elseif, conditions) {
        return elseif + '(' + conditions.replace(/\$/g, '') + ') {';
      })
      .replace(/endif/g, '}');
    // $name => name
    str = str.replace(/^\s*=\s*\$/g, '=');
    return str;
  }
});

Can see from the above code, the map() method to some of the rules of statements made some replacement, will eventually replace special syntax for nodetpl liked, so as to realize the final statement parsing.
