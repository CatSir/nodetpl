nodetpl methods

  • nodetpl.config: basic configuration, not necessary
  • nodetpl.get: get a template or precompiled js file
  • nodetpl.render: rendering a template
  • nodetpl.exec: js code execution inside the template, less used

NodeTplClass method

  • tpl.render: rendering the template

nodetpl.render(tplcontent[, data][, callback]);

Use data, direct rendering template code tpl content.

This method does not contain precompile process, compilation executed by the browser, typically used in artificial splicing or <script type="text/template"... scene.

nodetpl.render is an asynchronous method, which gives the results rendered in the callback function, different from tpl.render.

HTML:

<script id="favor-tpl" type="text/template">
<h1><?=@title?></h1>
<ul>
  <?for(var i=0; i<@favor.length; i++){?>
    <li><?=i?>：<?=@favor[i]?></li>
  <?}?>
</ul>
</script>

JS

var data = {
  "title": "Favor",
  "favor": [
    "Football",
    "Basketball",
    "Table tennis",
    "Glass ball"
  ]
};
var content = document.getElementById('favor-tpl').innerHTML;
nodetpl.render(content, data, function(d){
  console.log(d);
});

Or, do not write HTML, do string concatenation directly in the js also:

var template = '\
<script id="favor-tpl" type="text/template">\
<h1><?=@title?></h1>\
<ul>\
  <?for(var i=0; i<@favor.length; i++){?>\
    <li><?=i?>：<?=@favor[i]?></li>\
  <?}?>\
</ul>\
</script>';
nodetpl.render(template, ...
