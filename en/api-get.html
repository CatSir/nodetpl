nodetpl methods

  • nodetpl.config: basic configuration, not necessary
  • nodetpl.get: get a template or precompiled js file
  • nodetpl.render: rendering a template
  • nodetpl.exec: js code execution inside the template, less used

NodeTplClass method

  • tpl.render: rendering the template

nodetpl.get(url, [data, ] callback)

This method is very powerful, it can directly load compiled files, you can load the template source code is compiled.

1. url has no extention, or extention name is .js

Loads the specified executable file template (see precompile, .js suffix may be omitted), and render the template by data, result back to the callback function.

  • url: compiled js file path
  • data: data into the template, if data is false, suspend rendering
  • callback: after the template compilation, the callback function is executed, with the rendered HTML result as a parameter

Browser Direct reference:

<script src="./js/nodetpl.min.js"></script>

AMD/CMD/CommonJs/ES6:

// No need to include the nodetpl library,
// require the compiled js files instead.
 
var data = {
  "title": "Favor",
  "favor": [
    "Football",
    "Basketball",
    "Table tennis",
    "Glass ball"
  ]
};
 
nodetpl.get('http://www.nodetpl.com/demo/tpls/1', data, function(d){
  // console.log(d);
});
 
var tpl = require('./tpls/1');
var html = tpl.render(data);
// console.log(html);

Have a tryView demo

Under the above AMD / CMD / CommonJs / ES6 development model, require(...) returns a NodeTplClass instance, which has a render() method (refer tpl.render), is used to render the template with the specified data, and different from nodetpl.render.

If you do not want to render template (suspended rendering) immediately, the incoming data needs to be set to false, callback incoming data is no longer a rendered html, but NodeTplClass instance:

//[NodeTplClass]
{
  render: function(data) {
    //...
  }
}

The call the render() method where you want:

nodetpl.get('http://www.nodetpl.com/demo/tpls/1', false, function(tpl){
  var data = {
    "title": "Favor",
    "favor": [
      "Football",
      "Basketball",
      "Table tennis",
      "Glass ball"
    ]
  };
  // Here tpl not a rendered code, but an object, use render(data) method to render the template
  // var d = tpl.render(data);
  // console.log(d);
});

tpl is a NodeTplClass instance, when you get the tpl object, you can at any scene, repeatedly calling tpl.render() method, passing different objects in order to achieve the purpose of the template reuse.

2. url is the source template file

Loads the specified template source file (not pre-compiled), and render with data, rendering the results back to the callback.

Usage above, except that the incoming url path here is the template source file location, typically .tpl or .html

var data = {
  "title": "Favor",
  "favor": [
    "Football",
    "Basketball",
    "Table tennis",
    "Glass ball"
  ]
};
nodetpl.get('http://www.nodetpl.com/demo/tpls/1.tpl', data, function(d){
  // console.log(d);
});
