预编译

为了让模板代码在浏览器端运行的更快、更节省内存，同时，我们需要更加方便地管理模板文件，我们通常将模板代码存为一个独立的文件（后缀为.tpl），然后在开发阶段进行预编译。

预编译后的文件不再是模板代码，它是一个可执行的 .js 文件，浏览器自然可以非常迅速的识别并运行它。

安装预编译工具

预编译工具需要 node.js 环境支持，如果你没有安装 node.js ，请先安装它。

在命令行中输入下面的语句，完成安装：

npm install nodetpl -g

基本语法

nodetpl <path> -r <relativePath>
  • path：模板文件的路径，或模板所在的目录路径
  • relativePath：模板文件相对于项目根路径的路径，以 http 或 / 开头

通常情况下，每一个模板文件都会拥有自身一个唯一ID，这个ID可以确保该模板在运行过程中全局唯一。在这里，我们使用模板引用的路径来表示，通常为相对于项目根路径的位置（http 开始的路径也是允许的），后缀为我们编译后的 .js 文件。

编译单个文件

首先，将我们的模板文件保存在某个目录中（通常为 tpls），模板文件扩展名为 .tpl

假定项目的根路径为：/usr/local/testproject，则文件相对于根路径的位置应为：/tpls/demo/abc.tpl，运行下面的口令：

nodetpl /usr/local/testproject/tpls/demo/abc.tpl -r /tpls/demo/abc.js

我们可以看到，在模板文件同一路径下，生成了一个 .js 文件，该文件就是编译后的可执行文件。

批量编译指定目录中的多个模板文件

假定项目的根路径为：/usr/local/testproject，则目录相对于根路径的位置应为：/tpls

命令行进入项目根目录 /usr/local/testproject，执行下面的语句，即可编译该目录下所有模板文件：

nodetpl ./tpls -r /tpls

模板编译之后，就可以使用 nodetpl.get() 方法引用了。
