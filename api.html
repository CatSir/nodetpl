nodetpl 方法

nodetpl.config(options)

配置 nodetpl 基本参数

config 不是必须的，一般不需要做任何配置

下面是可以配置的参数示例：

nodetpl.config({
  base: 'http://www.nodetpl.com/static/tpl/',   // 强制指定模板根目录
  vars: {
    root: '$("#~")'
  },
  openTag: '<?',
  closeTag: '?>',
  cache: true
});
  • base：模板所在的根路径，可以省略。若配置该参数，则使用 .get() 方法填写的路径会相对于该路径进行查找
  • vars：变量相关信息
    • root：模板 JS 代码中的 $ROOT 所代表的对象。默认为 jQuery 对象，即 $("#~")，若你使用的是其他类库，可以调整该参数的指向
  • openTag：模板定界符开始标签，默认为：<?
  • closeTag：模板定界符结束标签，默认为：?>
  • cache：是否开启缓存，默认开启

这里大部分是可以采用默认值的，只是在极少数的场景下，可能会涉及到调整 vars.root 的指向。

nodetpl.get(url, [data, ] callback)

加载指定的模板可执行文件（编译方式参见预编译），并用 data 渲染，渲染后的结果返回给 callback。

  • url：模板编译后的 js 文件路径
  • data：传入模板的数据
  • callback：模板编译后，执行的回调函数，回调函数会将编译后的 HTML 作为参数传入
var data = {
  "title": "个人爱好",
  "favor": ["足球", "篮球", "乒乓球", "琉璃球"]
};
nodetpl.get('http://www.nodetpl.com/demo/tpls/1.js', data, function(d){
  console.log(d);
});

nodetpl.render(tplcontent[, data]);

使用数据 data，直接渲染模板代码 tplcontent。

此方法不包含预编译环节，通常用在人为拼接或者 <script type="text/template"... 的场景，前面首页给出的DEMO即是如此。

var data = {
  title: '个人爱好',
  favor: ['足球', '篮球', '乒乓球', '琉璃球']
};
var content = document.getElementById('testid').innerHTML;
var html = nodetpl.render(content, data);
alert(html);
