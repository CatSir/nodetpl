多模板

似乎前面了解到的模板功能，已经能够满足我们大部分开发所需，但是假如一个模板包含非常多的业务场景，模板代码量将会显得巨大而臃肿，为了解决这个问题，nodetpl 以非常巧妙的方式，引入了多模板的概念。

nodetpl 是这样定义多模板场景的：假如一个模板，包含了非常多的业务逻辑，而每个业务逻辑都是较为独立的片断，那么我们就可以把整个模板划分为多块，每一块表示一个业务场景；

  • 多模板包含一个主模板（main）和多个子模板，主模板是整个模板的入口
  • 使用 <template name=""></template> 标签将每个单独的模板包起来
  • 各模板之间通过 include 关键字相互引用

多模板的基本结构

与常规模板不同，多模板是由主模板和子模板组成，每一个模板被包含在一个 <template> 标签中：

<template name="main">
  // 主模板的代码
</template>
<template name="subname-1">
  // 子模板一的代码
</template>
<template name="subname-2">
  // 子模板二的代码
</template>

每一个模板都有一个 name 属性，其中，主模板是整个模板的入口，name 值为 main，子模板的 name 可以根据自身需要任意命名。

main 入口

主模板的 template name 是 main，我们可以像书写常规模板一样来书写主模板内的代码。

为了方便理解，我们假设这样的业务场景：

  • 一个卡片区域，默认情况下，显示个人名片信息，我们定义为“浏览”状态
  • 当点击“修改”链接的时候，显示个人名片修改的相关表单字段，我们定义为“修改”状态
  • “修改”界面内，可以修改各项信息，点击保存按钮，则自动保存，并显示保存后的“浏览”界面

查看DEMO

因此，除了主模板入口外，我们需要额外设计一个“浏览”状态，和一个“修改”状态的模板。

<template name="main">
  <style>
  .title {
    font-size: 14px;
    font-weight: bold;
  }
  .content {
    padding: 10px;
  }
  </style>
  <div id="$ROOT">
    <div class="title">个人名片 [<a class="link-modify" href="javascript:;">修改</a>]</div>
    <div class="content"></div>
  </div>
  <script>
  var contentBox = $ROOT.find('.content');
  var viewHtml = include('view');
  contentBox.html(viewHtml);
  $ROOT.find('.title a.link-modify').on('click', function(){
    var editHtml = include('edit');
    contentBox.html(editHtml);
  });
  </script>
</template>

在 nodetpl 中，使用关键字 include 表示引入一个模板。

上面代码表示：

  • 加载主模板（main）的时候，将名字为 “view” 的子模板加载到主模板的 “contentBox” 容器中，作为“浏览”显示用
  • 当点击标题中的“修改”链接的时候，将名字为 “edit” 的子模板加载到主模板的 “contentBox” 容器中。

子模板

同样，我们可以像书写常规模板一样来书写子模板内的代码，但略微会有不同：

浏览状态

<template name="view">
  <style>
  ul li{
    border: 1px solid #ccc;
  }
  </style>
  <div id="$SUBROOT">
    <ul>
      <li>姓名：<?=name?></li>
      <li>性别：<?=gender?></li>
      <li>年龄：<?=age?></li>
    </ul>
  </div>
</template>

编辑状态

<template name="edit">
  <style>
  ul li{
    margin: 0 10px;
    background-color: #eee;
  }
  </style>
  <div id="$SUBROOT">
    <form action="">
      <ul>
        <li>姓名：<input type="text" name="name" value="<?=name?>" /></li>
        <li>性别：<input type="text" name="gender" value="<?=gender?>" /></li>
        <li>年龄：<input type="text" name="age" value="<?=age?>" /></li>
      </ul>
      <div class="form-actions">
        <button type="submit">保存</button>
      </div>
    </form>
  </div>
  <script>
  $SUBROOT.find('form').on('submit', function(){
    var name = $(this).find('input[name="name"]').val(),
      gender = $(this).find('input[name="gender"]').val(),
      age = $(this).find('input[name="age"]').val();
    $DATA.name = name;
    $DATA.gender = gender;
    $DATA.age = age;
    var viewHtml = include('view');
    $ROOT.find('.content').html(viewHtml);
    return false;
  });
  </script>
</template>

从上面代码可以看到，与主模板不同的是，子模板中的“根元素”，不再使用 “$ROOT” 表示，取而代之的是 “$SUBROOT” 关键字。
