nodetpl-plusbeta

无拘无束，nodetpl-plus 是专门为 ES6 而生的模板引擎，它能完美地支持 70% 以上的 ES6/ES7 语法。

有了 nodetpl-plus，在开发过程中再也不用拘泥于浏览器的兼容性，大胆开始你的 ES6 之旅吧！

模板示例

下面 nodetpl 模板中多处使用了 ES6 的语法，nodetpl-plus 能够完美支持。

import $ from 'jquery';
import { getAge, getGender } from './userProfile';
let abc = require('abc');

<template name="main">
<style>
h1{color: #f00;}
h2{font-size: 12px;}
</style>
<div id="$ROOT">
  <h1>hello <?=@name?></h1>
  <?for(let i=0; i<10; i++){?>
    <p><?=i?></p>
  <?}?>
</div>
<script>
let age = getAge('12345');
let gender = getGender('12345');
let arr = [1, 2, 3];
arr = arr.map(v => v + 1);
let p = new Promise(function(resolve, reject){
  //...
});
p.then(function(){
  //...
}).catch(function(e){
  //...
});
console.log(include('sub'));
</script>
</template>
<template name="sub">
<div id="$SUBROOT">
  <h3>test sub</h3>
</div>
</template>

将上面文件保存为 test.tpl，运行 nodetpl-plus-cli ./test.tpl --transform 试试看？

语法变更

在 nodetpl 的基础上，nodetpl-plus 为实现 ES6+ 的支持（strict mode），做了下面的调整：

  • 模板中的样式，对于根节点，需要使用 $ROOT 声明，如：$ROOT{font-size:12px;}，不再支持省略（原来可以直接这样写：{font-size:12px;}）
  • 模板中引用传入数据的某个属性，不能直接使用属性名，改为 “@属性名” 引用。注意例子中的 <h1>hello <?=@name?></h1>
  • 需要引入某个模块，直接在模板顶部使用 import 或 require 语法即可
  • 模板只能通过预编译才能使用，不再支持浏览器端实时编译
  • 浏览器端不再引用任何 nodetpl 解释文件，生成的内容为标准的 JavaScript 代码
  • 直接通过 require 语句引入模板文件，调用 render 方法即可取得渲染后的数据

引用

通过 nodetpl-plus-cli 编译之后，可以像加载一般模块一样来引入。

引入的模板拥有一个 render() 方法，用来接收传入的数据以编译模板：

var tplMessage = require('./tpls/test2');
//...
var result = tplMessage.render(data);
console.log(result);

安装

安装 nodetpl-plus-cli

npm install nodetpl-plus-cli -g

安装完成后，运行 nodetpl-plus-cli -h，可以看到下面的帮助信息：

<?  NodeTplES Precompile Tools  ?>

  version: v1.2.3

  grammar:
    nodetpl-plus-cli <path> --extname .tpl --transform --library <library> --watch

  arguments:
    <path>     : template file or directory path.
    --extname  : -e, the template extention name, default is ".tpl".
    --transform: -t, transform code to es5 happy, default is false.
    --library  : -l, the output library, umd | amd | cmd | commonjs | var | es, default is commonjs
    --watch    : -w, watch the file, automatically compile if changed.

参数

<path>

要编译的模板文件路径，或模板文件所在的目录

-e or --extname

模板文件的扩展名，默认为“.tpl”

-t or --transform

是否将代码编译成 es5 可以识别的代码，默认为 false

-l or --library

指定模板的编译格式，可用值有：umd | amd | cmd | commonjs | var | es，默认为 commonjs

-w or --watch

是否监听文件变化，如果设置成 true，则当模板文件改动时，自动编译该模板

示例

编译某个文件

nodetpl-plus-cli ../test/test1.tpl
>> File: /Volumes/data/git/test/test1.tpl
>> [1]  /Volumes/data/git/test/test1.js

编译某个目录

nodetpl-plus-cli ../test
>> Directory: /Volumes/data/git/test
>> [1]  /Volumes/data/git/test/test1.js
>> [2]  /Volumes/data/git/test/test2.js

监听文件变化

nodetpl-plus-cli ../test --watch

将文件编译成 ES5 可以识别的语法

nodetpl-plus-cli ../test --transform

编译成 AMD 模块

nodetpl-plus-cli ../test --library amd

打包

目前 nodetpl-plus 支持输出 commonjs、CMD、AMD 等模块，可以完美地通过 webpack 等工具进行打包，同时还能用在 node 端，下面是一个使用 webpack 打包的示例：

安装 nodetpl-loader

npm install nodetpl-loader --save

配置 webpack

loaders: [{
  test: /\.(jsx?|tpl)$/,
  loader: 'babel',
  ...
}],
preLoaders: [{
  test: /\.tpl?$/,
  loader: 'nodetpl'
}]

User Express.js

手动配置 Express 的模板解析器

app.set('view engine', 'tpl');
app.set('views', path.join(__dirname, '../', 'views'));
app.engine('tpl', function(tplpath, data, callback) {
  if (typeof data === 'function') {
    data = callback;
    callback = null;
  }
  if (typeof callback !== 'function') {
    callback = function() {};
  }
  try {
    callback(null, require(tplpath.replace(/\.[^.]+$/, '')).render(data));
  } catch (e) {
    callback(e);
  }
});

添加 nodetpl-plus-cli 到当前项目：

npm install nodetpl-plus-cli --save-dev

修改 package.json

...
  "scripts": {
    "build": "nodetpl-plus-cli ./views --library node --transform",
    "postinstall": "npm run build",
  }
  ...

当模板有变动时，需要重新编译模板：

npm run build
