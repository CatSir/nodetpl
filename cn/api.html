nodetpl 方法

nodetpl 的方法有下面几个：

  • nodetpl.config：基本配置，非必须
  • nodetpl.get：获取某个模板
  • nodetpl.render：渲染某个模板
  • nodetpl.exec：执行模板里面的 js 代码，较少使用

常用的方法仅有 get() 和 render()，掌握了这两个方法，你也便掌握了 nodetpl。

nodetpl.config(options)

配置 nodetpl 基本参数

config 不是必须的，一般不需要做任何配置

下面是可以配置的参数示例：

nodetpl.config({
  base: 'http://www.nodetpl.com/static/tpl/',   // 强制指定模板根目录
  openTag: '<?',
  closeTag: '?>',
  cache: true
});
  • base：模板所在的根路径，可以省略。若配置该参数，则使用 .get() 方法填写的路径会相对于该路径进行查找
  • openTag：模板定界符开始标签，默认为：<?
  • closeTag：模板定界符结束标签，默认为：?>
  • cache：是否开启缓存，默认开启

这里大部分是可以采用默认值的，所以该方法可以省略。只是在极少数的场景下，你可能根据自己的需要做调整。

nodetpl.get(url, [data, ] callback)

该方法具有双面性，它可以直接加载编译后的文件，也可以加载模板源代码进行编译。

  • 指定 url 没有后缀，或后缀为 .js

    加载指定的模板可执行文件（编译方式参见预编译，后缀 .js 可以省略），并用 data 渲染，渲染后的结果返回给 callback。

    • url：模板编译后的 js 文件路径
    • data：传入模板的数据，若 data 为 false，则暂缓渲染
    • callback：模板编译后，执行的回调函数，回调函数会将编译后的 HTML 作为参数传入
    var data = {
  "title": "个人爱好",
  "favor": ["足球", "篮球", "乒乓球", "琉璃球"]
};
nodetpl.get('http://www.nodetpl.com/demo/tpls/1.js', data, function(d){
  // console.log(d);
});

    试一试查看DEMO

    若为 “暂缓渲染”，则 callback 传入的数据不再是渲染后的 html，而是下面结构体：

    {
  render: function(data) {...}
  toString: function() {...}
}

    在需要渲染的地方，直接执行该参数的 render() 方法即可：

    nodetpl.get('http://www.nodetpl.com/demo/tpls/1.js', false, function(tpl){
  var data = {
    "title": "个人爱好",
    "favor": ["足球", "篮球", "乒乓球", "琉璃球"]
  };
  // 这里的 tpl 不是渲染后的代码，而是一个对象，使用 render(data) 方法可以渲染模板
  // var d = tpl.render(data);
  // console.log(d);
});

  • 指定 url 为对应的模板源文件

    加载指定的模板源文件（未经过预编译），并用 data 渲染，渲染的结果返回给 callback。

    • url：模板源文件路径
    • data：传入模板的数据，若 data 为 false，则暂缓渲染
    • callback：模板加载后，执行的回调函数，回调函数会传入一个包含 render() 及 toString() 方法的对象

    用法同上，但不同的是这里传入的 url 路径是模板源文件地址，通常为 .tpl 或 .html

    var data = {
  "title": "个人爱好",
  "favor": ["足球", "篮球", "乒乓球", "琉璃球"]
};
nodetpl.get('http://www.nodetpl.com/demo/tpls/1.tpl', data, function(d){
  // console.log(d);
});

    试一试查看DEMO

    当得到 tpl 之后，你可以在任何场景下，反复调用 tpl.render() 方法，传入不同对象，以达到模板复用的目的。

    如果你想单纯的读取模板的源文件，直接嗲用 tpl.toString() 方法即可。

nodetpl.render(tplcontent[, data][, callback]);

  • 使用数据 data，直接渲染模板代码 tplcontent。

    此方法不包含预编译环节，通常用在人为拼接或者 <script type="text/template"... 的场景。

    HTML部分：

    <script id="favor-tpl" type="text/template">
<h1><?=title?></h1>
<ul>
  <?for(var i=0; i<favor.length; i++){?>
    <li><?=i?>：<?=favor[i]?></li>
  <?}?>
</ul>
</script>

    JS部分：

    var data = {
  title: '个人爱好',
  favor: ['足球', '篮球', '乒乓球', '琉璃球']
};
var content = document.getElementById('favor-tpl').innerHTML;
nodetpl.render(content, data, function(d){
  console.log(d);
});

    或者，不写在 HTML 中，直接在 js 中做字符串拼接亦可：

    var template = '\
<script id="favor-tpl" type="text/template">\
<h1><?=title?></h1>\
<ul>\
  <?for(var i=0; i<favor.length; i++){?>\
    <li><?=i?>：<?=favor[i]?></li>\
  <?}?>\
</ul>\
</script>';
nodetpl.render(template, ...

    试一试查看DEMO

  • 如果你只是想先得到模板，暂时不作渲染，在适合的场景下才去渲染，你可以将 render() 方法传入的 data 设置成 false 即可。

    在这种场景下，得到的结果是这样的一个结构体：

    {
  render: function(data) {...}
  toString: function() {...}
}

    在需要的场景下，直接调用该对象的 render() 方法即可：

    var tpl = nodetpl.render(content, false);
var html = tpl.render(data);
console.log(html);

    这种机制在 nodetpl 中称为 “暂缓渲染”。

nodetpl.exec(content);

执行 html 中的 JavaScript，该方法在极少数场景下使用（例如原生 innerHTML），因为大多数 js 库已包含自动执行的方法。

例如下面一段 HTML，通过常规 innerHTML 添加到某个 DOM 中时，JavaScript 不会执行，在添加完之后手动运行一下 exec 方法即可。

<h1>hello</h1>
<script>
  alert(0);
</script>
nodetpl.exec(content);
