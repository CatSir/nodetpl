nodetpl 方法

  • nodetpl.config：基本配置，非必须
  • nodetpl.get：获取某个模板或编译后的模板文件
  • nodetpl.render：渲染某个模板
  • nodetpl.exec：执行模板里面的 js 代码，较少使用

NodeTplClass 方法

  • tpl.render：渲染模板

或许目前你还不知道该如何下手，那么接下来我们先看两种常见的开发模式：

常规模式开发（不推荐）

常规模式下开发，我们需要引入 nodetpl 引擎，然后在需要的地方，直接通过 get() 方法调用模板文件：

浏览器直接引用：

<script src="./js/nodetpl.min.js"></script>

AMD/CMD/CommonJs/ES6：

var nodetpl = require('nodetpl');
 
nodetpl.get('http://www.domain.com/tpls/1.tpl', {
  title: '个人爱好'
}, function(d) {
  // console.log(d);
});

但是，实际开发过程中，我们一般并不会这么做，为了提高模板执行性能，我们往往先把模板进行预编译，然后再对该可执行文件进行引用。

预编译高性能开发（推荐）

先使用 预编译 工具对模板进行预编译，然后直接调用编译后的模板可执行文件。

浏览器直接引用：

<script src="./js/nodetpl.min.js"></script>
nodetpl.get('http://www.domain.com/tpls/1', {
  title: '个人爱好'
}, function(d) {
  // console.log(d);
});

AMD/CMD/CommonJs/ES6：

//不需要引入 nodetpl 库，直接引入编译后的 js 文件即可
var tpl = require('./tpls/1');
var html = tpl.render({
  title: '个人爱好'
});
// console.log(html);

上面调用方式能带来非常大的性能提升，所以作者非常推荐对模板进行预编译。
