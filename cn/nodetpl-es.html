nodetpl-esbeta

无拘无束，nodetpl-es 是专门为 ES6 而生的模板引擎，它能完美地支持 70% 以上的 ES6/ES7 语法。

有了 nodetpl-es，在开发过程中再也不用拘泥于浏览器的兼容性，大胆开始你的 ES6 之旅吧！

模板示例

下面 nodetpl 模板中多处使用了 ES6 的语法，nodetpl-es 能够完美支持。

import $ from 'jquery';
import { getAge, getGender } from './userProfile';

<template name="main">
<style>
h1{color: #f00;}
h2{font-size: 12px;}
</style>
<div id="$ROOT">
  <h1>hello <?=@name?></h1>
  <?for(let i=0; i<10; i++){?>
    <p><?=i?></p>
  <?}?>
</div>
<script>
let age = getAge('12345');
let gender = getGender('12345');
let arr = [1, 2, 3];
arr = arr.map(v => v + 1);
let p = new Promise(function(resolve, reject){
  //...
});
p.then(function(){
  //...
}).catch(function(e){
  //...
});
console.log(include('sub'));
</script>
</template>
<template name="sub">
<div id="$SUBROOT">
  <h3>test sub</h3>
</div>
</template>

语法变更

在 nodetpl 的基础上，nodetpl-es 为实现 ES6 的支持，做了下面的调整：

  • 模板中引用传入数据的某个属性，不能直接使用属性名，改为 @属性名 引用。注意例子中的 <h1>hello <?=@name?></h1>
  • 需要引入某个 ES 模块，直接在模板中使用 import 语法即可

特别说明

目前 nodetpl-es 只支持输出 commonjs 模块，可以完美地通过 webpack 等工具进行打包，暂时还不支持其他格式，在未来的版本中会给予支持。

安装

安装 nodetpl-es-cli

npm install nodetpl-es-cli -g

安装完成后，运行 nodetpl-es-cli -h，可以看到下面的帮助信息：

<?  NodeTplES Precompile Tools  ?>

  version: v1.0.0

  grammar:
    nodetpl-es-cli <path> --extname .tpl --es5 --watch

  arguments:
    <path>   : template file or directory path.
    --extname: the template extention name, default is ".tpl".
    --es5    : transform es6 code to es5, default is false.
    --watch  : watch the file, automatically compile if changed.

参数

<path>

要编译的模板文件路径，或模板文件所在的目录

--extname

模板文件的扩展名，默认为“.tpl”

--es5

是否将 es6 代码编译成 es5 代码，默认为 false

--watch

是否监听文件变化，如果设置成 true，则当模板文件改动时，自动编译该模板

示例

编译某个文件

nodetpl-es-cli ../test/test1.tpl
<?
  Hello, NodeTpl-ES
  extname: .tpl
  es5    : false
  watch  : false
  version: 1.0.0
?>
>> File: /Volumes/data/git/test/test1.tpl
>> [1]  /Volumes/data/git/test/test1.js

编译某个目录

nodetpl-es-cli ../test
<?
  Hello, NodeTpl-ES
  extname: .tpl
  es5    : false
  watch  : false
  version: 1.0.0
?>
>> Directory: /Volumes/data/git/test
>> [1]  /Volumes/data/git/test/test1.js
>> [2]  /Volumes/data/git/test/test2.js

监听文件变化

nodetpl-es-cli ../test --watch

将文件编译成 ES5 的语法

nodetpl-es-cli ../test --es5

引用

编译完成之后，就可以像加载一般模块一样来进行引入了。

引入的 es 模块拥有一个 render() 方法，用来传入数据编译模板：

import a from './test/test1';
var b = require('./test/test2');
//...
var result = a.render(data);
console.log(result);
