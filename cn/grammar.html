基本语法

HTML部分

在模板中的 HTML 部分，使用定界符“<?”和“?>”作为语法的开始和结束。

在定界符内，可以书写任意JavaScript语句，如：

<?for(var i=0; i<10; i++){?>
  <p>hello, world</p>
<?}?>

试一试查看DEMO

使用等于号“=”来输出对应的值：

<?for(var i=0; i<10; i++){?>
  <p>hello, world, 第 <?=i?> 次</p>
<?}?>

试一试查看DEMO

直接使用“@属性名”来表示传入的对象对应的属性。

假如传入的对象为：

{
  "title": "我是传入的标题",
  "favor": ["足球", "篮球", "乒乓球", "琉璃球"]
}

在模板中，可以通过 @title、@favor 来分别引用传入的 title、favor 值：

<h1><?=@title?></h1>
<ul>
  <?for(var i=0; i<@favor.length; i++){?>
    <li><?=i?>：<?=@favor[i]?></li>
  <?}?>
</ul>

XSS安全防范

“=” 默认会经过 XSS 安全过滤，以确保输出到页面中的内容是安全的，例如：

{
  "title": "<script>alert(0);</script>",
}
<h1><?=@title?></h1>

最终编译的结果是：

<h1>&lt;script&gt;alert(0);&lt;/script&gt;</h1>

如果你不想过滤 XSS，你可以使用 “==” 来代替 “=”，如：

<h1><?==@title?></h1>

将会输出：

<h1><script>alert(0);</script></h1>

CSS部分

nodetpl 允许模板定义自身的 CSS 样式，并且你大可不必为命名冲突而担忧，因为 nodetpl 有一套解决样式继承冲突的机制。

若要书写模板样式，你需要为模板的 HTML 部分定义一个 id 为 “$ROOT” 的根元素，之后直接在模板中书写样式即可：

<style>
  h1 {
    font-size: 20px;
    color: #f00;
  }
  p {
    text-indent: 2em;
  }
</style>
<div id="$ROOT">
  <h1><?=@title?></h1>
  <?for(var i=0; i<10; i++){?>
    <p>hello, world, 第 <?=i?> 次</p>
  <?}?>
</div>

试一试查看DEMO

诶？逆天了，这样写岂不把整个页面的样式都要覆盖掉？

没关系，nodetpl 不会容忍这样低级错误的发生，最终，nodetpl 会在输出之前，将每一行样式前面都添加“#ROOT”限定，这样就不会引发样式继承冲突了。

例外：若需要定义 “$ROOT” 的样式，则需要书写 “$ROOT” 限定符，如：

$ROOT {
  border: 1px solid #f00;
}

JavaScript部分

nodetpl 允许模板调用自身的 JavaScript 脚本，以方便你做一些事件绑定相关的工作。

在 JavaScript 代码中，nodetpl 为你预置了下面几个变量：

  • ROOT：根节点 DOM 元素，即：document.getElementById('根节点')
  • $DATA：传入的数据

同样，你也不需要为命名冲突而担忧，因为 nodetpl 会让你的脚本在一个匿名函数中执行。

<div id="$ROOT">
  <h1><?=@title?></h1>
  <?for(var i=0; i<10; i++){?>
    <p>hello, world, 第 <?=i?> 次</p>
  <?}?>
</div>
<script>
var padding = '20';
ROOT.style.padding = padding + 'px';
// OR use jQuery
$(ROOT).css('color', '#f00');
console.log($DATA.title);
</script>

试一试查看DEMO

没错，虽然这里你定义了一个 padding 变量，但是它并不是在 window 对象中执行的，因此不会污染到 window 。
