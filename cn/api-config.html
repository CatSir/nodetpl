nodetpl 方法

  • nodetpl.config：基本配置，非必须
  • nodetpl.get：获取某个模板或编译后的模板文件
  • nodetpl.render：渲染某个模板
  • nodetpl.exec：执行模板里面的 js 代码，较少使用

NodeTplClass 方法

  • tpl.render：渲染模板

nodetpl.config(options)

配置 nodetpl 基本参数

config 不是必须的，一般不需要做任何配置

下面是可以配置的参数示例：

nodetpl.config({
  base: 'http://www.nodetpl.com/static/tpl/',   // 强制指定模板根目录
  openTag: '<?',
  closeTag: '?>',
  strict: true,
  map: function(str) {
    return str;
  }
});
  • base：模板所在的根路径，可以省略。若配置该参数，则使用 .get() 方法填写的路径会相对于该路径进行查找
  • openTag：模板定界符开始标签，默认为：<?
  • closeTag：模板定界符结束标签，默认为：?>
  • strict：是否开启严格模式，默认开启
  • map: 传递一个解析函数（转换器），用来解析其他语法，例如 php、jsp 等

对于参数 strict，如果不启用严格模式，则模板中引用数据时的“@”可以省略，如：

<h2><?=@title?></h2>

在非“strict mode”下可以书写为：

<h2><?=title?></h2>

但，随着 ES6 的发展，我们并不推荐这样。

这里大部分是可以采用默认值的，所以该方法可以省略。只是在极少数的场景下，你可能根据自己的需要做调整。

转换器：功能强大的 map

nodetpl 允许你传递你喜欢的任意语法，你可以根据需要，传递 map() 方法，告诉 nodetpl 该如何解析这些非常规的语句，下面是一个简单的例子：

假如有下面的模板：

<div>
  <?foreach ($array as $key => $value):?>
    hello, <?=$key?> <?=$value?>
  <?endforeach?>
  <?for($i=0; $i<=10; $i++):?>
    hello, <?=$i?>
  <?endfor?>
  <?if($age < 20):?>
    hello, young man.
  <?endfor?>
</div>

很显然，nodetpl 并不知道该如何处理这种语法，但是我们可以通过 map() 告诉 nodetpl，遇到这些语句时，该怎么处理：

nodetpl.config({
  strict: false,
  map: function(str) {
    // foreach => arr.forEach
    str = str
      .replace(/foreach\s*\(\s*(\$array)\s+as\s+(\$key)\s*=>\s*(\$value)\s*\):/g, function(all, arr, key, value) {
        return arr.replace(/^\$/g, '') + '.forEach(function(' + value.replace(/^\$/g, '') + ', ' + key.replace(/^\$/g, '') + ') {';
      })
      .replace(/endforeach/g, '});');
    // for
    str = str
      .replace(/for\s*\(\s*([^;]+)(.*)\):/g, function(all, vars, conditions) {
        return 'for(var ' + vars.replace(/\$/g, '') + conditions.replace(/\$/g, '') + ') {';
      })
      .replace(/endfor/g, '}');
    // if
    str = str
      .replace(/((?:else\s+)?if)\s*\((.*)\):/g, function(all, elseif, conditions) {
        return elseif + '(' + conditions.replace(/\$/g, '') + ') {';
      })
      .replace(/endif/g, '}');
    // $name => name
    str = str.replace(/^\s*=\s*\$/g, '=');
    return str;
  }
});

从上面的代码可以看到，map() 方法对语句中的一些规则做了一些替换，最终将特殊的语法替换为 nodetpl 喜欢的书写方式，从而实现最终的语句解析。
