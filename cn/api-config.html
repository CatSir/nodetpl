nodetpl 方法

  • nodetpl.config：基本配置，非必须
  • nodetpl.get：获取某个模板或编译后的模板文件
  • nodetpl.render：渲染某个模板
  • nodetpl.exec：执行模板里面的 js 代码，较少使用

NodeTplClass 方法

  • tpl.render：渲染模板

nodetpl.config(options)

配置 nodetpl 基本参数

config 不是必须的，一般不需要做任何配置

下面是可以配置的参数示例：

nodetpl.config({
  base: 'http://www.nodetpl.com/static/tpl/',   // 强制指定模板根目录
  openTag: '<?',
  closeTag: '?>',
  strict: true
});
  • base：模板所在的根路径，可以省略。若配置该参数，则使用 .get() 方法填写的路径会相对于该路径进行查找
  • openTag：模板定界符开始标签，默认为：<?
  • closeTag：模板定界符结束标签，默认为：?>
  • strict：是否开启严格模式，默认开启

对于参数 strict，如果不启用严格模式，则模板中引用数据时的“@”可以省略，如：

<h2><?=@title?></h2>

在非“strict mode”下可以书写为：

<h2><?=title?></h2>

但，随着 ES6 的发展，我们并不推荐这样。

这里大部分是可以采用默认值的，所以该方法可以省略。只是在极少数的场景下，你可能根据自己的需要做调整。
