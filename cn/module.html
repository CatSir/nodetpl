多元化支持

nodetpl 能够支持常规开发及各种模块化开发，你可以根据自身项目需求，选择任意一种开发模式。

普通模式的开发比较简单，直接在页面中引入 nodetpl 的 js 文件即可，下面来了解一下模块化开发的使用方式。

CMD（seaJs）

seaJs 基本配置

配置 seaJs 基本参数，增加 nodetpl 的 alias 项：

seajs.config({
  alias: {
    'nodetpl': 'lib/nodetpl.client.min'
  }
});

通过 require 将 nodetpl 模块引入：

define(function(require, exports, module) {
  var nodetpl = require('nodetpl');

  nodetpl.get('/demo/tpls/1', {
    title: '个人爱好',
    favor: ['足球', '篮球', '乒乓球', '琉璃球']
  }, function(d) {
    // d
  });
});

AMD（requireJs）

requireJs 基本配置

配置 requireJs 基本参数，增加 nodetpl 的 paths 项：

require.config({
  paths: {
    'nodetpl': 'lib/nodetpl.client.min'
  }
});

通过 require 将 nodetpl 模块引入：

define(function(require, exports, module) {
  var nodetpl = require('nodetpl');

  nodetpl.get('/demo/tpls/1', {
    title: '个人爱好',
    favor: ['足球', '篮球', '乒乓球', '琉璃球']
  }, function(d) {
    // d
  });
});
