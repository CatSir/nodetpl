多元化支持

nodetpl 能够支持常规开发及各种模块化开发，你可以根据自身项目需求，选择任意一种开发模式。

普通模式的开发比较简单，直接在页面中引入 nodetpl 的 js 文件即可，下面来了解一下模块化开发的使用方式。

CMD（seaJs）、AMD（requireJs）

不执行预编译的情景（不推荐）：

通过 require 将 nodetpl 模块引入：

define(function(require, exports, module) {
  var nodetpl = require('nodetpl');
  var data = {
    title: '个人爱好',
    favor: ['足球', '篮球', '乒乓球', '琉璃球']
  };
  nodetpl.get('/demo/tpls/1.tpl', data, function(d) {
    // d
  });
});

模板预编译后，无需再引用 nodetpl：

define(function(require, exports, module) {
  //var nodetpl = require('nodetpl');
  var data = {
    title: '个人爱好',
    favor: ['足球', '篮球', '乒乓球', '琉璃球']
  };
  var tpl = require('./demo/tpls/1');
  var result = tpl.render(data);
  console.log(result);
});

试一试查看DEMO

通过 require 的方式为同步引入，这是执行效率最高的一种方式。若想改为异步引入，你仍需要引入 nodetpl 主文件，然后通过 nodetpl.get 方法引入：

define(function(require, exports, module) {
  var nodetpl = require('nodetpl');
  var data = {
    title: '个人爱好',
    favor: ['足球', '篮球', '乒乓球', '琉璃球']
  };
  var tpl = nodetpl.get('./demo/tpls/1', data, function(d) {
    console.log(d);
  });
});

CommonJs（node.js）

node.js 调用方式与前面 AMD/CMD 基本一致：

var tpl = require('./demo/tpls/1');
var data = {
  title: '个人爱好',
  favor: ['足球', '篮球', '乒乓球', '琉璃球']
};
var result = tpl.render(data);
console.log(result);
