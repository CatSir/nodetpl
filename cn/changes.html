升级日志

4.5.02016.06.28
  • 新增 map() 方法，允许用户自定义模板语法，可以解析类似 php、jsp 等大多数语言
4.3.02016.06.15
  • 修复了模板文件后缀不是 .tpl 时编译工具编译出错的bug
4.2.12016.06.06
  • 应部分用户要求，针对 AMD/CMD 模式，支持同步调用（require）的基础上，继续支持异步调用（nodetpl.get），使用者可以根据自身情况来决定同步或异步
4.1.02016.06.06
  • 增加对 XSS 注入的安全过滤，引入新语法 “==”
  • 预编译工具增加对 gbk 编码格式的支持，可以通过传递 encoding 参数进行设置
4.0.02016.06.03
  • 重构大部分代码，优化渲染机制，提高渲染性能
  • css 中，根元素选择器不能再使用空白表示，如：{dispay:block;} 应当写成 $ROOT{display:block;}
  • 模板中的属性引用，不再推荐直接书写属性名，需要在属性名前面添加“@”，如：<?=@title?>，但非 strict 模式下仍然允许
  • 调整了命令行工具相关输入参数名称
  • AMD/CMD/CommonJs/ES6 模式下，可以不依赖 nodetpl 直接运行
  • 优化了模板中js代码的算法
(p)2.0.02016.4.22
  • 发布 nodetpl-plus 稳定版，在 nodetpl 的基础上，语法做了微调，功能做了跨域性提升：支持 ES6 语法，支持 webpack 打包
3.3.22015.12.30
  • 优化 node 版本的 require 功能，支持相对路径引用
3.3.12015.12.28
  • 优化 node 版本，解决了传参异常导致应用崩溃的bug
3.3.02015.12.26
  • 优化 node 版本，模板文件可以运行在沙箱中
  • 修复了 exporess.js 使用 nodetpl 时的若干 bug
3.0.02015.12.22
  • 优化了模块化加载方式（CMD/AMD），提高编译性能
  • $ROOT / $SUBROOT 废弃，若使用 jQuery，请用 $(ROOT) 和 $(SUBROOT) 代替重要变更
  • config.vars 属性废弃
2.3.32015.12.04
  • get() 方法增强，支持直接读取模板代码，暂缓渲染
2.3.12015.12.02
  • 增加 exec 方法
  • 完善 seaJs 支持，提高模块化场景下代码的运行效率
  • render 方法增加同步、异步两种方式

    常规用法（异步）：

    nodetpl.render(..., data, function(d){
  // 这里是编译结果
});

    同步方法（不安全，考虑适配更多场景，不支持使用此种渲染方法，请使用异步）：

    var html = nodetpl.render(..., data);
2.2.62015.11.30
  • 完善 <?=..?> 功能，自动判断引用的变量是否 undefined
2.1 to 2.2
  • 编译工具做重大升级，不再要求输入 -o 属性，支持文件监听，支持指定模板后缀
2.0 to 2.1
  • 关键字 “@” 可以省略，在将来的版本中将要废弃
1.x to 2.0
  • 支持通过 config 灵活配置引擎开关
  • 内置 _fixcss 方法即将废弃，使用 1.x 版本编译的模板需要重新编译
  • 正式支持 CMD 模块化开发
  • 多模板引用由 “require” 关键字改为 “include”，原 “require” 关键字废弃
  • 增加关键字 ROOT，代表原生 DOM 节点
  • 默认 $ROOT 指向为 jQuery 对象（选择器 $("#~") ），如需使用其它类库，请手动 config 设置 
    nodetpl.config({
  vars: {
    root: '...'
  }
});
